Nykaa App
Redesign

A comprehensive redesign of India's leading beauty e-commerce platform — elevating product discovery, brand storytelling, and checkout experience for 25M+ users.

01 — Challenge
The Problem

Nykaa's app had accumulated significant feature debt. Navigation was cluttered, visual language was inconsistent across categories, and the checkout funnel was dropping users at multiple touchpoints. The goal: modernise the end-to-end experience without alienating a deeply loyal user base.

Hero Screens — Home, Product Detail & Cart
02 — Process
How I Worked
01
Discovery & UX Audit
Deep audit of existing screens — catalogued every inconsistency, measured drop-off rates at each funnel stage, and benchmarked against Sephora, Purplle, and Myntra Fashion.
02
User Research & Persona Mapping
Interviewed 20+ active Nykaa users. Identified three core personas: the Deal Hunter, the Brand Loyalist, and the First-Timer — each with distinct navigation intent.
03
Information Architecture Rebuild
Reduced primary navigation tabs from 6 to 4. Moved low-engagement items behind progressive disclosure to significantly cut cognitive load on first open.
04
Visual Design System
Token-based design system in Figma — 4 brand tiers, 120+ components, refined type scale and luminosity-extended brand palette.
05
Prototype & Dev Handoff
High-fidelity interactive prototypes for all key flows. Annotated component specs in Figma Dev Mode. Full engineering handoff documentation.
Product Grid — Category View
Product Detail — Add to Cart
03 — Solution
Key Design Decisions

Three pillars drove every decision: Discoverability — a search-first home with contextual suggestions; Trust — verified reviews, transparent ingredients, shade-match AR; and Speed — a 2-tap checkout for returning users, cutting average purchase time by 40%.

The visual language shifted from dense promotional banners to an editorial grid — letting product photography breathe while maintaining Nykaa's signature magenta system.

The Problem

The previous dashboard was disorganised — data scattered across disconnected views with no visual hierarchy. The challenge was to create clarity and visual order while strictly honouring the client's brand guidelines.

Admin Dashboard — Main Analytics View
02 — Process

03

Kargo
plex

What Travelocity did for travel, Kargoplex is doing for global cargo. An AI-driven platform consolidating shipments, cutting costs, and delivering real-time visibility — built entirely from scratch.

01 — Challenge
The Problem

The global cargo industry was fragmented — shippers, forwarders, and ports operated on completely disconnected systems. Kargoplex needed a unified platform built from zero that could bring all three user groups onto a single, intuitive interface. With no existing product to reference, every UX pattern had to be invented and validated.

Shipper Dashboard — Live Shipment Tracking
02 — Process
User-Centred Approach
01
Research & Pain Point Analysis
Extensive research into logistics platforms, freight management tools, and cargo tracking software. Mapped pain points for each of the three distinct user groups.
02
Wireframes & IA
Detailed wireframes establishing clear information hierarchy for shippers, forwarders, and admin. Shared early with the client and engineering for feasibility alignment.
03
High-Fidelity Responsive Design
Built the full product across desktop, tablet, and mobile simultaneously — developing the design system in parallel for consistent component reuse across all breakpoints.
04
Prototyping & Iteration
Continuous prototype-feedback cycles with client and developer input. Complex flows — booking, tracking, dispute management — each went through 4–6 iteration rounds.

03 — Solution
The Platform

Kargoplex connects shippers directly to forwarders through a clean booking flow. Users track shipment status in real-time, communicate with forwarders directly, and manage all documentation in one place. The AI matching system — surfaced through a recommendation card pattern — suggests optimal forwarders based on route, cargo type, and cost.
